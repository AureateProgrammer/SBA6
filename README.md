The biggest challenge was definitely the CSS layout and responsiveness. Getting the result cards to look good on both a desktop and an iPhone required a lot of work with Flexbox. I ran into an issue where the long URLs would overflow and break the card's container. I solved this by using justify-content: space-between and setting specific overflow rules so the text would wrap or truncate instead of pushing the "Copy" button off the screen. Another hurdle was the GitLab Pages deployment, where I had to troubleshoot the .gitlab-ci.yml file to ensure the project correctly routed to index.html.

Handling the "wait time" for the TinyURL API was the most critical part of the logic. I used async/await to make sure the app didn't try to render a result card before the shortened link actually existed. To keep the app from crashing on bad inputs, I wrapped the fetch request in a try/catch block. If a user enters a broken link or the API is down, the catch block triggers a clean alert message instead of letting the whole script fail. I also used encodeURIComponent to ensure that special characters in the URLs didn't break the API request.
